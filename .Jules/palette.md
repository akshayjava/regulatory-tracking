## 2024-05-18 - Missing ARIA Labels in Unlabeled Form Inputs and Improved Empty States
**Learning:** This app uses inline styles heavily and inputs like Search and Status filters did not have associated `<label>` tags. This caused accessibility issues for screen readers. Additionally, replacing generic "No results found" messages with a clear recovery path (e.g., a CTA to clear filters) significantly improves the user experience.
**Action:** Next time working on custom styled inputs, ensure ARIA attributes are added if standard label tags aren't present. Also, look out for empty states that leave the user stranded, and try to add actionable buttons to help them recover.\n## 2024-05-24 - Inline Styles and Focus Visibility\n**Learning:** In a React application heavily utilizing inline styles (without CSS classes or styled-components), pseudo-classes like `:focus-visible` cannot be used directly in the `style={{}}` prop. This leads to severe accessibility issues as interactive elements with `outline: 'none'` become completely invisible to keyboard users.\n**Action:** When working in this specific codebase, manually implement focus rings by attaching `onFocus` and `onBlur` event handlers to semantic elements (`<button>`, `<input>`, `<select>`) to toggle a `boxShadow` (e.g., `boxShadow: '0 0 0 2px #60a5fa'`).
