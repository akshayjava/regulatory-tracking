## 2024-04-24 - Inline Styles and Focus Management
**Learning:** This application heavily relies on inline styles (`style={{...}}`) which actively hide default focus outlines (`outline: 'none'`). Because CSS pseudo-classes like `:focus-visible` cannot be natively applied via inline styles, components can easily become completely inaccessible to keyboard users.
**Action:** When adding or modifying interactive elements (inputs, buttons) that use inline styles, manually handle accessibility focus states by attaching `onFocus` and `onBlur` handlers to manage an `isFocused` state. Use this state to conditionally render custom focus rings (e.g., via `boxShadow` and border color changes) to ensure keyboard navigation remains visible and intuitive. Always add an `aria-label` to inputs if they lack a visible associated label.
