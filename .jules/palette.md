## 2024-04-30 - Focus Ring Accessibility with Inline Styles
**Learning:** In React applications that rely heavily on inline styles without a CSS framework or external stylesheet, it's impossible to use pseudo-classes like `:focus-visible` natively via `style={{}}`. This leads to poor keyboard accessibility since interactive elements (like tabs or buttons) lack visual focus indicators.
**Action:** Implemented a reusable pattern attaching `onFocus` and `onBlur` handlers to semantic interactive elements (e.g. `<button>`, `<input>`, `<select>`) to render custom focus rings via inline `outline` or `boxShadow` styles, and removed native `outline: 'none'` conflict where necessary. Also documented this constraint to ensure all future interactive elements use this manual focus management approach.
