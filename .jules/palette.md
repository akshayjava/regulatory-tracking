## 2024-04-26 - Keyboard Accessible List Items with Inline Styles
**Learning:** Using `div` tags with `onClick` handlers for list items (like the "Next Steps" checklist) prevents keyboard navigation (Tab to focus, Enter/Space to select) and is inaccessible to screen readers. Furthermore, since the codebase uses inline styles, pseudo-classes like `:focus-visible` are not supported, meaning default or customized focus rings are lost.
**Action:** Always use semantic `<button>` elements for interactive, actionable items instead of `<div>`. To handle focus states when using inline styles, attach `onFocus` and `onBlur` event listeners to dynamically apply and remove CSS properties like `boxShadow` to simulate a focus ring. Adjust styles like `width: '100%'`, `textAlign: 'left'`, and `outline: 'none'` to retain the original appearance.
