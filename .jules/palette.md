## 2024-05-18 - Handling Focus States with React Inline Styles
**Learning:** In applications like this one where external CSS or utility classes are heavily avoided in favor of inline `style={{...}}`, CSS pseudo-classes such as `:focus-visible` cannot be applied natively. This causes serious accessibility issues as interactive elements naturally lose their visual focus ring when default browser outlines are removed (e.g. `outline: 'none'`).
**Action:** Implemented a reusable pattern attaching `onFocus` and `onBlur` event handlers directly to semantic elements (like `<button>`, `<input>`, and `<select>`). These handlers update a local state variable (e.g., `focusedElement`), which is then used within the inline `style` prop to dynamically render a custom focus ring (such as `boxShadow: '0 0 0 2px #60a5fa'`).
